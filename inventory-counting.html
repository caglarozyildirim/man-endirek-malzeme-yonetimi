Database Schema - Inventory Counting

INV_CountingHeader:

  • inventory_counting_header_id: PK, Unique counting session ID
  • correction_type: Counting | Recount | Spot Check
  • creation_date_time: Session creation timestamp
  • created_by_user_id: FK to Users (creator)
  • created_for_user_id: FK to Users (responsible person)
  • storage_id: FK to Storage
  • location: Location code (optional filter)
  • bin: Bin code (optional filter)
  • status: In Progress | Completed | Cancelled
  • description: Session description

INV_CountingItem:

  • inventory_counting_item_id: PK
  • inventory_counting_header_id: FK to Header
  • item_number: Line item sequence number
  • material_id: FK to Materials
  • stock_record_id: FK to StockRecords (snapshot reference)
  • stock_quantity_system: System quantity at time of count
  • unit_of_measure: PC, M, KG, L
  • currency: TRY, USD, EUR
  • unit_value: Unit cost/value
  • storage, location, bin: Item location details
  • counting_results: Actual counted quantity (user input)
  • counted_by_user_id: FK to Users (who counted)
  • counted_date_time: When counted
  • difference: counting_results - stock_quantity_system
  • difference_value: difference √ó unit_value
