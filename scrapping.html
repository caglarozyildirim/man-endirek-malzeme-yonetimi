üìã Ekran Bilgisi: Stok Duzeltme - Hurda ve Diger Islemler

Excel Spec: 24 IC - SO - Scrapping or Other

Amac: Kullanilamaz hale gelen malzemelerin stoktan dusurulmesi ve diger stok duzeltme islemleri.

Pattern: Document-based - Modal ile belge olusturma, liste gorunumu, genisletilebilir satirlar

Data Structure Alanlari (Stock Correction Header):

  • Belge: stock_correction_document_id, document_title, document_description, correction_type
  • Kullanici: created_by_user_info, created_for_user_info, creation_datetime
  • Deger: euro_total_value, try_total_value
  • Onay 1: approver_1_user_id, 1_approved_by_user_id, 1_approved_datetime, 1_approval (bool)
  • Onay 2: approver_2_requirement_check (bool), 2_approved_by_user_id, 2_approved_datetime, 2_approval (bool)
  • Onay 3: approver_3_requirement_check (bool), 3_approved_by_user_id, 3_approved_datetime, 3_approval (bool)
  • Gruplar: comment_group_id, document_group_id

Data Structure Alanlari (Stock Correction Item):

  • Kalem: stock_correction_document_id, item_number, stock_record_id, selected_quantity

Screen Designs Ek Alanlari (Goruntulenecek):

  • Malzeme: material_number, material_title, unit_of_measure, currency, unit_value, total_stock_value
  • Lokasyon: cost_id, storage, location, bin
  • Acik Islemler: open_material_requests, open_srm_requests, open_purchasing_orders
  • Ek Bilgi: attachment_group_id, attachment_id, attachment_type, attachment_link
  • Goods Movement: goods_movement_id, stock_quantity, material_group, accounting_related

Duzeltme Tipleri:

  • Hurda: Kullanilamaz hale gelen malzemeler
  • Hasar: Fiziksel hasar gormus malzemeler
  • Kalite Hatasi: Kalite kontrolunden gecemeyen malzemeler
  • Son Kullanma: Raf omru dolmus malzemeler
  • Diger: Obsolete, kayip vb. durumlar

Duzeltme Yontemleri (Correction Method):

  • Stoktan Dusme: Malzeme stoktan tamamen dusurulur
  • Deger Duzeltme: Malzeme degeri guncellenir (amortisman)
  • Transfer: Hurda deposuna transfer edilir

Veritabani Islemleri - Data Structure IMMA.xlsx Uyumlu

API Endpoint: POST /api/stock-correction

Tablo Yapisi (Stock Correction Header - Data Structure'dan):

Stock_Correction_Header:
  - stock_correction_document_id (PK, int 8)
  - stock_correction_document_title (text 64)
  - stock_correction_document_description (text 64)
  - correction_type (char 1)
  - creation_datetime (datetime)
  - created_by_user_info (int 8, FK to User)
  - created_for_user_info (int 8, FK to User)

  /* Onay Seviye 1 - Her zaman gerekli */
  - approver_1_user_id (int 8, FK to User)
  - 1_approved_by_user_id (int 8, FK to User)
  - 1_approved_datetime (datetime)
  - 1_approval (bool)

  /* Onay Seviye 2 - 5000-25000‚Ç∫ arasi icin gerekli */
  - approver_2_requirement_check (bool)
  - 2_approved_by_user_id (int 8, FK to User)
  - 2_approved_datetime (datetime)
  - 2_approval (bool)

  /* Onay Seviye 3 - 25000‚Ç∫ uzeri icin gerekli */
  - approver_3_requirement_check (bool)
  - 3_approved_by_user_id (int 8, FK to User)
  - 3_approved_datetime (datetime)
  - 3_approval (bool)

  /* Degerler */
  - euro_total_value (float 14, 12 int 2 float)
  - try_total_value (float 14, 12 int 2 float)

  /* Grup Referanslari */
  - comment_group_id (int 8, FK to Comment_Group)
  - document_group_id (int 8, FK to Document_Group)

Tablo Yapisi (Stock Correction Item - Data Structure'dan):

Stock_Correction_Item:
  - stock_correction_document_id (int 8, FK to Stock_Correction_Header)
  - stock_correction_document_item_number (int 4, Sira no: 1, 2, 3...)
  - stock_record_id (int 8, FK to Material_Stock)
  - selected_quantity (float 15, 12 int 3 float)

Screen Designs'tan Ek Goruntulenecek Alanlar (Join ile):

/* Material tablosundan */
  - material_number (text 13)
  - material_title (text 64)
  - material_group (FK to Material_Group)
  - unit_of_measure (FK to Unit_of_Measure)

/* Material_Stock tablosundan */
  - stock_quantity (float 15)
  - cost_id (int 8, FK to Cost)

/* Cost tablosundan */
  - storage (FK to Storage_Type)
  - location (text)
  - bin (text)

/* Hesaplanan Degerler */
  - currency (TRY/EUR)
  - unit_value (birim fiyat)
  - total_stock_value (secilen miktar x birim fiyat)
  - euro_total_value (EUR karsiligi)
  - try_total_value (TRY karsiligi)

/* Acik Islemler (Sorgu ile) */
  - open_material_requests (Acik talep sayisi)
  - open_srm_requests (Acik SRM sayisi)
  - open_purchasing_orders (Acik PO sayisi)

Iliskili Tablolar:

Document (Attachment):
  - document_id (PK)
  - document_group_id (FK)
  - attachment_type
  - attachment_link
  - created_by_user_info
  - creation_datetime

Comment_Group:
  - comment_group_id (PK)
  - (Yorumlar burada gruplanir)

Comment:
  - comment_id (PK)
  - comment_group_id (FK)
  - entered_by_user_id
  - comment_text
  - creation_datetime

Is Akisi Kurallari

  1. Belge Olusturma: Modal ile belge bilgileri girilir (baslik, tip, aciklama)
  2. Malzeme Ekleme: Belge olusturulduktan sonra kalemler eklenir
  3. Deger Hesaplama: FIFO prensibi ile birim ve toplam degerler hesaplanir
  4. Onay Sureci:
    • 0 - 5.000 TL: Depo Sorumlusu onayi yeterli
    • 5.000 - 25.000 TL: Depo Muduru onayi gerekli
    • 25.000 TL uzeri: Finans Direktoru onayi gerekli
  5. Stok Dusurme: Onay sonrasi stok miktarlari guncellenir
  6. Muhasebe Entegrasyon: Stok deger dususu muhasebe sistemine aktarilir
